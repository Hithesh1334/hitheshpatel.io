Built three production MCP (Model Context Protocol) servers from scratch — for JIRA, Slack, and Snowflake. Core idea: give LLMs structured, controlled access to engineering tools without letting them improvise. Each server exposes purpose-built tools so the model triggers an action, not a reasoning chain.

Tools designed so the LLM triggers actions — control and logic stay on the engineering side, not in the model's reasoning
MCP Resources expose live data (Snowflake schemas, JIRA context, Slack channel lists) as grounded context — not just API calls
User-configurable safety hooks block DROP, DELETE, TRUNCATE at the server layer regardless of what the model generates
Full engineering loop in one agent session: read ticket → query Snowflake → write code → post Slack update → open PR
